## ⚠️ Anomalies techniques détectées

### 1. Pas de header HTTP `Last-Modified`
Aucune page ne renvoie de `Last-Modified`. C'est un signal de fraîcheur absent pour Googlebot.
→ **Action** : Ajouter `Last-Modified` basé sur la date de modification réelle du fichier ou du `updated` frontmatter.

### 2. Canonical auto-référencé correct mais sans signal fort
Les canonicals sont présents et corrects, mais sans `Last-Modified` ni sitemap `lastmod` cohérent avec le contenu réel (toutes les pages affichent `lastmod: 2026-07-14` même si inchangées).

### 3. Densité publicitaire desktop
2 placeholders Ezoic + 3 blocs AdSense sur la homepage. Sur desktop (écran large), ces pubs occupent visuellement plus de place que sur mobile.
→ **Action** : Vérifier le layout desktop — les pubs sont-elles au-dessus de la ligne de flottaison ? Font-elles du layout shift ?

### 4. `content/blogg/` vs `content/blog/`
Le sitemap référence `/blogg/*` et le contenu est dans `content/blog/`. Le dossier `content/blogg/` ne contient qu'un dossier `prompts/`. **Pas de conflit critique détecté** mais la clarification recommandée par le workflow (juillet 2026) n'est pas faite.
